# Cognitive Co-Processor Skill

This skill provides the procedural knowledge required to work within the Cognitive Co-Processor Framework. It focuses on externalizing cognitive functions to mitigate "ADHD Leak" and optimize "Cognitive IOPS".

## Core Architecture

The framework is divided into three infrastructure layers:

1.  **The Surface (Interface):** Optical interface for spatialized I/O. Remediation for sequential string processing deficits.
2.  **The Engine (Execution):** The harness for deterministic tool orchestration and asynchronous execution.
3.  **The Soil (Context):** Persistent memory bank for structured state management. Remediation for working memory leaks.

## Key Workflows

### 1. Spatial Mapping (Surface)
When presenting data or status, convert linear syntax into "visual furniture". 
- **Action:** Use quadrants or L-shape motor mapping to represent agent states or logic branches.
- **Reference:** See `references/architecture.md` for details on spatial anchors.

### 2. Pressure Mapping (Engine)
Analyze data outside of a linear timeline to bypass sequential processing bottlenecks.
- **Action:** Utilize 3D simulation and historical architectural analysis before execution.

### 3. Cognitive IOPS Optimization (Soil)
Maximize external state persistence to preserve "Mental RAM".
- **Action:** Explicitly commit all transient variables, syntax strings, and metadata to structured local storage (e.g., DuckDB, OpenViking).
