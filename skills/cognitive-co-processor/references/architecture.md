# Cognitive Co-Processor Architecture Reference

## 1. The Surface (Interface)
**Target Deficit:** Sequential String Processing (rote syntax reading, linear data ingestion).
**Remediation:** Optical Interface using Spatial Mapping.

- **Visual Furniture:** Converting abstract text into spatially anchored objects.
- **TARS Social Modulation:** I/O tone filter (Humor 60%, Honesty 95%) isolated at the interface layer.
- **Tools:** VSCode (Visual-spatial logic maps), Google Antigravity.

## 2. The Engine (Harness)
**Target Deficit:** Sequential Processing Bottlenecks / Time-bound Logic.
**Remediation:** Execution Loop using Pressure Mapping.

- **Pressure Mapping:** Analyzing historical/architectural data non-linearly using 3D simulations.
- **Deterministic Action:** Translating simulations into sandboxed agent actions.
- **Tools:** Claude Code, Gemini CLI, Ollama.

## 3. The Soil (Context)
**Target Deficit:** Working Memory ("ADHD Leak").
**Remediation:** Persistent Memory Bank for Structured State.

- **Cognitive IOPS:** Success metric defined by the rate of external state commit/recall.
- **Metadata Persistence:** Offloading "Mental RAM" (variable names, syntax strings) to local storage.
- **Tools:** OpenViking (Repository metadata), DuckDB (Local querying).
